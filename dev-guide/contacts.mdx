---
title: "Contact ID"
description: "Use contact_id to group SketricGen conversations under users or contacts from your own system."
---

Contact ID connects SketricGen conversations to a stable user or contact identifier from your own system.

Where to go next:

- If you are using the REST API, SDKs, AI Workforce, or a custom product integration, see [Public API: Contact ID and User Grouping](/dev-guide/public-api-usage#contact-id-and-user-grouping).
- If you are adding the chat bubble or iframe to a website, see [Deploy an AI Chatbot to a Custom Website](/deploy/custom-website#identify-known-users-with-contact-id).

In short:

- Public API requests use `contact_id` in the JSON body.
- Website widgets use `data-contact-id` on the script tag.
- iFrame and fullscreen links use the `contactId` query parameter.

Use a stable internal ID, UUID, or hashed value. Avoid emails, phone numbers, names, or other directly identifiable personal data.
