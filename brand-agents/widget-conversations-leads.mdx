Brand Agents are built to be published. The widget and conversation tools help you test, launch, and improve the customer-facing experience.

## Widget and Fullscreen Modes

A Brand Agent can be shown as:

- A website chat widget
- A fullscreen chat page
- An iframe embed where supported

Customize the visual experience so it matches your brand. Check the greeting, colors, logo, callouts, and any CTA shown to visitors.

## Public Visibility

The agent must be published/public for a client-side widget to work. The widget cannot safely include a private API key in browser code.

If the widget does not load, confirm the Brand Agent is published and that the embed snippet points to the correct agent.

## Test Before Launch

Use realistic visitor prompts:

- "What do you do?"
- "How much does this cost?"
- "Can I book a demo?"
- "Where can I find your refund policy?"
- "Can I talk to a human?"

If the agent gives vague answers, improve the Knowledge Base or tighten the instructions.

## Conversations

Conversations show what visitors asked and how the Brand Agent responded. Use conversations to find:

- Missing knowledge
- Confusing answers
- Lead qualification opportunities
- Out-of-scope questions
- Tool or connector errors

## Leads

Leads show visitor details captured through the Lead Capture tool. Review lead records with the original conversation context so your team understands what the visitor wanted.

Use this feedback loop: review conversations and leads, improve instructions and knowledge, then republish.
